# DisasterLens nickname synonym graph.
#
# Source-of-truth for the `disasterlens_nicknames` synonyms set loaded into Elastic
# Cloud Serverless via the Synonyms API (PUT _synonyms/disasterlens_nicknames).
# Referenced by every `name_standard` analyzer in data/mappings/*.json.
#
# Format: Solr-style equivalence groups. One group per line. Comma-separated tokens
# are treated as fully equivalent (search for any → matches all). Comments start with #.
#
# Each given name belongs to exactly ONE group — Spanish↔English calques are merged
# into the same group as their nicknames, so {Michael,Mike,Miguel,Miguelito} is one
# class rather than two overlapping ones.
#
# Coverage rationale (PRD §7 stress cases):
#   • Spanish diminutives + Sp↔En calques  — Carlos/Carlitos/Charles, John/Juan/Johnny
#   • English nicknames                    — Catherine/Kate/Cathy, William/Bill/Will
#   • Arabic romanisation                  — Mohammed/Muhammad/Mohamed/Mohd
#   • Vietnamese romanisation              — handled by name_translit analyzer (ICU),
#                                            not here; name-order inversion is an
#                                            application-layer concern
#
# Expand judiciously: every nickname group widens recall and erodes precision.
# When in doubt, add the case to the eval set first and verify the gain.
